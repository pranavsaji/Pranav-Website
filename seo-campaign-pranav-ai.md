## Phase 1 — On-page (done in code / quick wins)

The technical foundation (schema, sitemap, robots, metadata) is already excellent. Remaining on-page levers:

- [ ] Strengthen `knowsAbout` + `alternateName` in Person schema to reinforce the AI-security entity.
- [ ] Add an FAQ block ("Who is Pranav Saji?", "What does Pranav Saji do in AI?") with FAQPage schema — these win the People-Also-Ask box and reinforce the name+AI association.
- [ ] Ensure every blog post has the author linked to the Person entity (`author: {@id: #pranav-saji}`) so each post feeds your entity authority.
- [ ] Internal linking: every blog post links back to /about and to 2-3 sibling posts with descriptive anchor text ("Pranav Saji's analysis of...").
- [ ] One pillar page: `/ai-security` — a hub titled around "Pranav Saji — AI Security" that links to all security posts. Pillar pages rank far better than scattered posts.

## Phase 2 — Content engine (the real ranking driver)

Google ranks entities that keep producing relevant content. You already publish security posts — systematize it:

- **Cadence:** 2 posts/month minimum, every one tagged AI security / LLM security.
- **Title formula:** lead with the topic, you are the author signal. Each new post = a new indexable page targeting new long-tail queries that all reinforce "Pranav = AI security."
- **Repurpose:** every blog post → LinkedIn article + LinkedIn newsletter + dev.to + Medium, each linking back to pranav-saji.com with anchor text "Pranav Saji". (You already have all these profiles in your schema.)

## Phase 3 — Backlinks + off-page (what actually moves the needle)

Rankings for a name are 80% off-page. Priorities, highest leverage first:

1. **Press / bylines** — your `media-article/` pipeline IS your SEO strategy. Every HackerNoon / Dark Reading / VentureBeat byline that links to pranav-saji.com is a top-tier backlink. Keep pitching. Ensure the byline bio links to the site with "Pranav Saji" anchor text.
2. **Wikipedia/Wikidata** — you have a Wikidata item (Q139970723). Enrich it: occupation = AI security researcher, employer, education, notable work. Wikidata feeds Google's Knowledge Graph directly.
3. **Speaking** — Sessionize profile is good. Each conference page that lists you + links back is a strong, topical backlink.
4. **Guest podcasts / interviews** on AI security → show-notes backlinks.
5. **GitHub** — pin AI-security repos, put pranav-saji.com in your profile and repo READMEs.
6. **Directory/citation consistency** — same name, title, photo, URL everywhere (Crunchbase, Qwoted, LinkedIn). Consistency strengthens the entity.

## Phase 4 — Measurement (free, do this weekly)

- **Google Search Console** — verification token already in your code. Submit sitemap. Watch the "Performance" tab for impressions/clicks on `pranav ai`, `pranav ai security`, etc. This is your only real scoreboard.
- **Bing Webmaster Tools** — import from GSC in one click.
- Track monthly: position for the target queries table above. Celebrate movement, not absolute rank.

## What NOT to do

- Don't buy backlinks or "SEO packages" from cold emailers — Google penalizes link schemes; it can get you de-ranked.
- Don't keyword-stuff "Pranav AI" unnaturally — modern Google reads it as spam.
- Don't expect overnight results. Entity authority builds over 6-18 months of consistent signals.
